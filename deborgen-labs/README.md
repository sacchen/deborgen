# Labs

Structured exercises for building deborgen and understanding it deeply.

## The rule

Don't use AI to write implementation code. You can ask for hints, ask why
something isn't working, ask for an explanation of a concept — but the code
you submit should be code you typed and understand.

If AI writes it for you, the lab is over and the learning stops.

## How each lab works

Each lab has:
- **Concept** — the idea you're building toward
- **Task** — exactly what to build
- **Acceptance criteria** — how you know you're done
- **Hints** — pointers to the right direction, not the answer

Don't open the hints until you've tried. Being stuck for 20 minutes is normal
and productive. Being stuck for 2 hours means ask for a hint.

## Sequence

| Lab | Title | Language |
|-----|-------|----------|
| 01  | Phase A tests | Python |
| 02  | Code reading: the coordinator | Python |
| 03  | Go worker: heartbeat loop | Go |
| 04  | Go worker: claim and execute jobs | Go |
| 05  | Go worker: artifacts | Go |

Do them in order. Each lab assumes the previous one.

## Before starting Lab 03

You need Go installed. Check with `go version`. If not installed:
https://go.dev/dl/

You do not need any Go experience. Lab 03 starts from scratch.
